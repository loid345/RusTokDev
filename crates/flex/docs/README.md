# flex docs

`flex` is the extraction target for Flex attached-mode shared contracts.

Current scope:
- registry contracts (`FieldDefinitionService`, `FieldDefRegistry`)
- transport-agnostic command/view DTOs
- generic CRUD orchestration helpers for registry-driven operations (`list/find/create/update/deactivate/reorder`)

Roadmap and sequencing are tracked in `docs/architecture/flex.md` (Phase 4.5).
