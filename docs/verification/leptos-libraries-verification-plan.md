# План верификации Leptos-библиотек (rolling)

- **Статус:** В процессе
- **Режим:** Непрерывная верификация в параллели с общей разработкой
- **Частота полного прогона:** 1 раз в неделю (full verification агентами)
- **Цель:** Гарантировать, что функционал реализуется через библиотечные контракты и не утекает в обходные реализации в приложениях

---

## 1. Объект верификации

### 1.1 Leptos-библиотеки

- [ ] `crates/leptos-auth`
- [ ] `crates/leptos-forms`
- [ ] `crates/leptos-zustand`
- [ ] `crates/leptos-graphql`
- [ ] `crates/leptos-shadcn-pagination`
- [ ] `crates/leptos-ui`
- [ ] `crates/leptos-zod`
- [ ] `crates/leptos-table`
- [ ] `crates/leptos-hook-form`

### 1.2 Важное уточнение

Библиотеки развиваются одновременно с продуктовой разработкой. Поэтому этот план — **rolling-процесс**, а не разовая кампания.

---

## 2. Единый чеклист (применяется к каждой библиотеке)

### A. Контракт и зона ответственности

- [ ] Подтверждён актуальный контракт библиотеки (public API, обязательные сценарии).
- [ ] Подтверждён список потребителей (`apps/admin`, `apps/storefront`, при необходимости `apps/next-*`).
- [ ] Проверено, что обязательные сценарии проходят через библиотеку.

### B. Поиск bypass-реализаций

- [ ] Найдены обходы библиотечных контрактов в приложениях.
- [ ] На каждый обход создана задача переноса в библиотеку.
- [ ] Зафиксированы временные обходы с дедлайном устранения.

### C. Проверка ошибок и регрессий

- [ ] Проведена проверка на баги/регрессии в ключевых сценариях библиотеки.
- [ ] Критические дефекты получили owner и приоритет.
- [ ] Добавлены/обновлены тесты или сценарии воспроизведения (если требуется).

### D. Проверка добавления нового функционала

- [ ] Проверено, что новый функционал за период добавлен в библиотеку (или одновременно с её расширением).
- [ ] Проверено, что app-слой остаётся orchestration-слоем, а не местом постоянной реализации библиотечных задач.
- [ ] Для новых capability отмечено, какие приложения уже мигрировали на библиотечный путь.

### E. Актуализация документации

- [ ] Обновлён `README.md` библиотеки (контракт, примеры, ограничения).
- [ ] Обновлены локальные docs внутри соответствующего `crates/*`/`apps/*` при изменении поведения.
- [ ] Обновлён `docs/index.md`, если изменились ссылки/структура документации.
- [ ] Если изменение затрагивает UI-контракты/дизайн-систему, обновлены документы в `docs/UI/` (Internal UI workspace).

---

## 3. Weekly full verification (агенты)

### 3.1 Ритм

- [ ] Раз в неделю выполняется полный проход по всем Leptos-библиотекам.
- [ ] Дополнительно выполняется внеплановый точечный проход по библиотекам, которые менялись в течение недели.

### 3.2 Выходные артефакты weekly-прогона

- [ ] Статус по каждой библиотеке: `✅/🟡/❌`.
- [ ] Список найденных bypass-реализаций.
- [ ] Список ошибок/регрессий.
- [ ] Список новых функций за неделю и место реализации (library path vs app path).
- [ ] Статус актуализации документации.
- [ ] Рекомендации на следующую итерацию.
